The unusual part is the combination.

Truthmark does not need to claim that no other tool has one piece of this. The stronger point is that these properties work together in one repository-native loop.

01

Day-to-day work does not depend on a CLI, MCP server, hook, or daemon.

The installed workflow is carried by repository files and host-native agent surfaces. Optional helpers can validate and refresh, but normal agent work should not be blocked by a missing service.

  • Fits Codex, Claude Code, Copilot, Cursor, OpenCode, and similar hosts.
  • Keeps the fallback path inspectable in Git.
02

Truth docs are curated continuously.

Truthmark treats documentation as a closeout responsibility for code changes. If a doc becomes too broad, the answer is ownership repair, not just more prose.

  • Truth Sync updates current behavior.
  • Truth Structure splits broad or mixed-owner docs.
03

Product and engineering truth stay in separate lanes.

Product capability promises and engineering implementation facts are related, but they are not the same document. Truthmark keeps that distinction reviewable.

  • Product docs describe promises and boundaries.
  • Engineering docs describe current implementation behavior.

Separate lanes make the docs more honest.

A product promise should not be buried inside implementation prose. An implementation fact should not accidentally become a product commitment.

Product lane What users can rely on

Product truth captures capabilities, boundaries, non-goals, acceptance criteria, and decisions.

  • “Operators can inspect failed webhook delivery history before replay.”
  • “Truthmark does not require a hosted service for normal repository workflow execution.”

Engineering lane What the code currently does

Engineering truth captures runtime behavior, contracts, operations, architecture, and implementation decisions.

  • “Backoff starts at 30 seconds and doubles per attempt.”
  • “The Pages workflow deploys only the committed static site under site/**.”

$ npm install -g truthmark
$ truthmark config
$ truthmark init
$ truthmark check
